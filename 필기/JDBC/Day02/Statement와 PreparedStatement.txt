


JDBC 에서 데이터베이스로 쿼리문을 전송할 때 사용하는 객체 2가지

1. Statement 방식
2. PreparedStatement 방식


## 공통점 ##
1. 두 객체 모두 SQL 질의문을 DBMS로 전달하는 역할
2. 사용시 반드시 try ~ catch문 또는 throws 처리를 해야 함

## 차이점 ##
Statement는 정적인 쿼리문만 처리할 수 있음
	즉, 쿼리문에 값이 미리 입력 되어 있어야 함

ex) *INSERT INTO TEST VALUES('id','pw','name');

PreparedStatement 방식은 동적인 쿼리문을 처리할 수 있음
-> SQL 문장이 미리 컴파일 되고, 실행 시간 동안 인수값을 위한 고악ㄴ을
	확보 할 수 있음
-> 각 각의 인수에 대해 위치홀더('?')를 사용하여 SQL 문장을 정의할 수 있게 해줌

ex) *INSERT INTO VALUES(?,?,?);

-> 이후에 setString 메소드를 통해 값을 셋팅할 수 있음

-> setStrung(1,'id');
-> setStrung(2,'pw');
-> setStrung(3,'name');
