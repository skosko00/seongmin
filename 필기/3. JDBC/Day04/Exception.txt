

Exception -> 예외

즉, 코드에 있어서 예외가 발생하게 되는것은 예외처리를 해주어야 함

기존 코드에서 보면 DAO클래스에서 SQL 처리시 발생하는 각종 예외에 대해
DAO에서 예외 에러 메시지를 출력하고 있음

하지만 실제적으로 DAO에서 예외가 발생한다 하여도 DAO에서 예외 메시지를
보내고 끝내기만 하면 실제 DAO를 호출한 Service 혹은 Controller에서는
해당 예외 상황을 알지 못한 상황이 됨

보통 예외처리는 해당 메소드를 최초로 호출한 쪽에서 처리 하는 것이 맞음
	-> 즉, 전체 조회의 경우 컨트롤러에서 전체조회 서비스를 요청하였기 때문에
	예외가 발생하면 컨트롤러에게 예외 상황에 대한 메시지가 발생하고
	처리할 수 있도록 해주는게 기본 비즈니스 로직에 알맞음
