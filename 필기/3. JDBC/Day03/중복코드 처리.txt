

## 중복 코드를 해결
-> 중복 되는 코드들이 발생하면, 중복되는 코드들은 하나로 코딩하여 처리 해야함

1. 드라이버 연결
2. DBMS 연결 (Connection)
3. close 코드
4. commit 과 rollback 코드

-> 중복 코드를 해결하였지만 계속된 new 연산자를 남발(메모리 낭비) 하게 되는
문제가 발생하고 있음
	- 해결책 : 싱글톤 방식을 사용하여 해결 하도록 하겠음

## SingleTon ##
-> 클래스에 대한 객체가 프로그램 구동 내내 한 개만 작성되어 사용될 수 있도록
하는 방식으로 메모리 낭비가 발생하지 않도록 하는 디자인 패턴
