

@ 공지사항(게시판 로직)

## 전체 리스트 불러오기 ##

1. notice 서블릿을 요청
2. 비즈니스 로직을 통해 전체 리스트를 읽어모
3. jsp 페이지를 통해 출력

※ 단, 페이징 처리를 하지 않게 되면 모든 공지사항이 한 페이지에 출력하게 되고,
그렇게 되면 스크롤바가 엄청나게 길어지게 됨



## 전체 리스트 페이징 처리 ##
-> 페이징 처리를 하기 위해서는 몇개의 값을 정하고 시작 해야 함
	- 한 페이지당 보이는 게시물의 개수
	- 현재 위치를 중심으로 시작 navi 부터 끝 navi가 구성되는 개수


1. notice 서블릿 요청
2. 서블릿 요청시 현재의 페이지 값을 셋팅 (처음 요청시에는 page는 1page로 셋팅)
3. Service에서 처리
	- DAO를 호출하기 전 2가지 값을 설정
	- 1. 한 페이지당 보이는 게시물의 개수
	- 2. 현재 위치를 중심으로 시작 navi 부터 끝 navi가 구성되는 개수

4. 첫번째 DAO 요청 - 현재 페이지 (요청한) 목록 처리
5. 두번째 DAO 요청 - navi 생성 처리

6. 2개의 값을 리턴 할 수 있도록 Page 정보를 담는 VO 생성 (ex. PageData)

7. 리턴 값을 Service에서 받아서 PageData VO에 정보를 채운 후 Controller로 리턴
8. Controller에서 jsp로 보냄
